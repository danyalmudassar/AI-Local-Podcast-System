## Getting Started

1. Fork the repository on GitHub
2. Clone your fork locally:
   ```bash
   git clone https://github.com/YOUR_USERNAME/local-podcast-agent.git
   cd local-podcast-agent
   ```
3. Follow the setup instructions in the [README](README.md) to get the project running locally

## Development Workflow

1. Create a new branch for your feature or bug fix:
   ```bash
   git checkout -b feature/my-new-feature
   # or
   git checkout -b bugfix/issue-description
   ```

2. Make your changes and commit them with descriptive commit messages:
   ```bash
   git add .
   git commit -m "Add feature: description of my new feature"
   ```

3. Push your changes to your fork:
   ```bash
   git push origin feature/my-new-feature
   ```

4. Open a pull request to the main repository

## Code Style

### Python
- Follow PEP 8 guidelines
- Use type hints where appropriate
- Write docstrings for functions and classes
- Keep functions and methods reasonably short

### JavaScript/TypeScript
- Use consistent indentation (2 spaces)
- Use meaningful variable and function names
- Follow existing code patterns in the project

### General
- Write clear, descriptive commit messages
- Keep pull requests focused on a single feature or bug fix
- Update documentation as needed

## Pull Requests

1. Ensure your PR addresses an issue or describes the changes clearly
2. Include any relevant issue numbers in the PR description
3. Update the README or documentation if necessary
4. Add tests if applicable
5. Make sure all tests pass
6. Keep PRs reasonably sized (preferably under 500 lines of changes)

## Issues

### Creating Issues
- Check existing issues before creating a new one
- Use a clear and descriptive title
- Provide detailed steps to reproduce bugs
- Describe the expected vs. actual behavior
- Include your environment information (OS, Python version, etc.)

### Issue Labels
We use labels to categorize issues:
- `bug`: Something isn't working
- `enhancement`: New feature or improvement
- `documentation`: Improvements or additions to documentation
- `good first issue`: Good for newcomers
- `help wanted`: Extra attention is needed

## Community

- Be respectful and inclusive
- Provide constructive feedback
- Help others when you can
- Follow the project's code of conduct

## Questions?

If you have questions about contributing, feel free to open an issue with the `question` label or reach out to the maintainers.

Thank you for contributing to Local Podcast Agent!
